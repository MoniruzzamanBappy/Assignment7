How does JAVASCRIPT work?

  • JavaScript is an interpreted language.
  • It is a client side scripting language which means it runs in the client server.
  • JS is single theaded language and for this reason it has single call stack and can do one thing at a time.
  • Its use JIT compilers to run the code.
  • In run time its allocates memory and exicutes the scripts.

    • What does JS event loop do?

  • In JS exicutable funtion are uses a message queue , that list is processed in run time and handled the queue by the processing time
  • It waits synchronously for a message to arrive and if one is not already available and waiting to be handled
  • Each and every message is processed completely before any other message is processed by the run time
  • In event loop js continiously run the process that monitors the callback queue and call stack.
  • In processing time if the call stack is not empty then the event loop is wait and when its get empty then next function is placed from callback queue, if callback queue is empty then nothing will happen.

    • Difference Between Local Storage And Session storage.

  • Local Storage data does not expire,but Session Storage data is deleted when the web page session ends.
  • Local Storage data deleted via javascript or manually and Session Storage data is stored only for the duration of a session.
  • Local Storage data survives over page reloads and restores but Session Storage data is cleared.
  • In new tab or window Local Storage data gets restored but in session storage data are duplicated from the previous one.
